韦东山的教学视频中采用的是Ubuntu虚拟机，而我这里采用了Win11自带的WSL2，至于为什么使用WSL2呢？其实也没啥就是给自己找麻烦，首先，WSL2的Ubuntu 22 里面会缺少些东西，如Tree之类的。好在你可以让自己学习使用apt包管理进行补全就是了，而且也不依赖于百问网已经配置好环境，即开即用的Ubuntu18。这里我选择了自己去配置环境，缺啥补啥就是了，而且在AI的加持下可以更快的掌握Linux的使用思路和逻辑。并且百问网教程其实也是有几年了，期间很多的指令在现在可能有一定区别，所以尝试在新一点的环境下学习还是有帮助的。



## 一、先确认板子端 `adbd` 是否启动

首先在开发板上执行：

```
ps | grep adbd
```

如果输出类似下面这样：

```
365 root     /usr/bin/adbd
```

说明开发板端的 `adbd` 已经启动。

但这里有一个很重要的认识：

**板子端 `adbd` 在运行，并不代表 WSL2 中的 Ubuntu 一定能识别到这块板子。**

因为主机侧还要经过 USB 识别、Windows 转发、WSL attach、权限配置这几层。

------

## 二、在 WSL 中安装必要工具

### 1. 安装 `lsusb`

我一开始在 WSL 中执行 `lsusb` 时，提示命令不存在。
 后来才知道 `lsusb` 来自 `usbutils` 包，所以先安装：

```
sudo apt update
sudo apt install -y usbutils
```

安装后就可以执行：

```
lsusb
```

------

### 2. 安装 adb

接着安装 adb 工具：

```
sudo apt install -y android-sdk-platform-tools android-sdk-platform-tools-common
```

检查是否安装成功：

```
which adb
adb version
```

如果能看到 adb 版本信息，说明主机侧 adb 已可用。

------

## 三、Windows 通过 `usbipd` 把 USB 设备挂到 WSL2

WSL2 默认不能像原生 Linux 一样直接访问 USB 设备，因此需要借助 `usbipd`。

在 Windows **管理员 PowerShell** 中执行：

```
usbipd list
```

找到开发板对应的 `BUSID` 后，依次执行：

```
usbipd bind --busid <BUSID>
usbipd attach --wsl --busid <BUSID>
```

这里我踩过一个很典型的坑：

很多时候设备显示为 `shared`，但 **shared 并不等于已经进入 WSL**。
 真正关键的是执行：

```
usbipd attach --wsl --busid <BUSID>
```

只有 attach 成功后，WSL 才能看到 USB 设备。

另外，如果开发板重启、USB 线重新插拔，或者 WSL 重启，这一步通常都要重新做一次。

------

## 四、在 WSL 中确认 USB 设备是否已经进入系统

在 Ubuntu 中执行：

```
lsusb
```

我成功识别时看到过类似输出：

```
Bus 001 Device 001: ID 1d6b:0002 Linux Foundation 2.0 root hub
Bus 001 Device 002: ID 1d6b:0104 Linux Foundation Multifunction Composite Gadget
Bus 002 Device 001: ID 1d6b:0003 Linux Foundation 3.0 root hub
```

其中最关键的是这一行：

```
ID 1d6b:0104 Linux Foundation Multifunction Composite Gadget
```

这说明开发板已经以 USB gadget 的形式进入 WSL。

如果 `lsusb` 中根本没有这块设备，那么问题通常不在 adb，而在：

- Windows 没识别到设备
- 没有执行 `usbipd attach --wsl`
- attach 之后设备又掉了
- USB 线或板子状态异常

------

## 五、使用 adb 查看设备状态

在 WSL 中执行：

```
adb kill-server
adb start-server
adb devices -l
```

我遇到过两种典型状态。

------

### 情况 1：设备已识别，但没有权限

输出类似这样：

```
100ask_IMX6ULL         no permissions (missing udev rules? user is in the plugdev group)
```

这个状态说明：

- 设备已经被 adb 识别到了
- 不是“完全认不到”
- 真正的问题是当前普通用户没有权限访问 USB 设备

这时候可以先用 `sudo` 验证一下：

```
sudo adb kill-server
sudo adb start-server
sudo adb devices -l
```

如果这时状态变成：

```
100ask_IMX6ULL         device
```

那就说明：

**USB 链路和 adb 本身都已经正常，问题只是权限。**

------

### 情况 2：设备正常

当看到类似下面的输出时，说明连接已经成功：

```
100ask_IMX6ULL         device usb:1-1 transport_id:1
```

这时候就可以正常执行：

```
adb shell
adb push test.txt /tmp/
adb pull /tmp/test.txt
```

------

## 六、解决 `no permissions` 问题

当设备已经被识别，但显示 `no permissions` 时，我是这样处理的。

### 1. 把当前用户加入 `plugdev` 组

```
sudo usermod -aG plugdev $USER
```

------

### 2. 添加 udev 规则

```
sudo tee /etc/udev/rules.d/51-android-100ask.rules >/dev/null <<'EOF'
SUBSYSTEM=="usb", ATTR{idVendor}=="1d6b", ATTR{idProduct}=="0104", MODE="0666", GROUP="plugdev"
SUBSYSTEM=="usb_device", ATTR{idVendor}=="1d6b", ATTR{idProduct}=="0104", MODE="0666", GROUP="plugdev"
EOF
```

------

### 3. 重载规则

```
sudo udevadm control --reload-rules
sudo udevadm trigger
```

------

### 4. 重启 WSL

在 Windows PowerShell 中执行：

```
wsl --shutdown
```

重新打开 Ubuntu 后，再执行：

```
id
adb kill-server
adb start-server
adb devices -l
```

如果 `id` 输出中已经包含 `plugdev`，同时 `adb devices -l` 中设备状态变成 `device`，那就说明问题已经解决。

------

## 七、设备“突然消失”时如何排查

我在调试过程中还遇到过一种情况：

前一刻设备还能识别，后一刻设备又消失了。

这类问题通常不是 adb 本身有问题，而是 USB 链路发生了变化，比如：

- 板子重启了
- 开发板重新枚举 USB
- 重新插拔了 USB 线
- WSL 的 attach 会话失效了

这时候建议固定按下面的顺序排查。

### 第一步：在 Windows 中检查设备状态

```
usbipd list
```

看设备是否还在列表中。

------

### 第二步：如果没 attach，就重新挂载到 WSL

```
usbipd attach --wsl --busid <BUSID>
```

如果设备没有 bind 过，也可以重新执行：

```
usbipd bind --busid <BUSID>
usbipd attach --wsl --busid <BUSID>
```

------

### 第三步：回到 WSL 中确认 USB 是否已进入系统

```
lsusb
```

如果 `lsusb` 里还是没有设备，那说明 USB 还没真正进 WSL。

------

### 第四步：再检查 adb

```
adb kill-server
adb start-server
adb devices -l
```

如果这时候又回到了 `no permissions`，那就说明 USB 已经恢复，只是权限又掉了，再回到前面的权限修复步骤即可。

------

## 八、我踩过的坑

### 1. `lsusb` 不存在

原因：没有安装 `usbutils`

解决：

```
sudo apt install -y usbutils
```

------

### 2. Windows 里设备已经 `shared`，但 WSL 里还是看不到

原因：只共享了设备，没有真正 attach 到 WSL

正确做法：

```
usbipd attach --wsl --busid <BUSID>
```

------

### 3. 误以为“识别不到”，其实是“识别到了但没权限”

当 `adb devices -l` 里出现：

```
no permissions
```

说明设备已经识别到，只是权限不对。

这种情况不要再去怀疑线、板子系统、内核配置，先处理权限问题。

------

### 4. 设备会“消失”

这通常不是故障，而是因为：

- 板子重启
- USB 重新枚举
- attach 状态失效

在 WSL2 + `usbipd` 场景里，这种现象很常见。
 排查思路一定要固定下来：先看 `usbipd list`，再看 `lsusb`，最后看 `adb devices -l`。
